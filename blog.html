  • Discuss the scope of var, let, and const

    • In JavaScript, `var` has function scope, allows hoisting, and lacks block scope.`let` and `const`are block-scoped, preventing hoisting.`let` allows variable reassignment, while `const` makes variables constant after initialization. Use `var`for broader scope,`let` for variables that may change, and `const` for constants.

  • Tell us the use cases of null and undefined

    • In JavaScript, `undefined` typically indicates the absence of a value or uninitialized variables. It's automatically assigned to variables that haven't been given a value, making it useful for checking if a variable exists or has a value. On the other hand, `null` represents the intentional absence of a value and is explicitly used to clear or indicate an empty value. It differs from `undefined`, which is automatically assigned and implies uninitialized variables. Both `undefined` and `null` play distinct roles in handling variables and values within JavaScript, with `undefined` often associated with default or missing values and `null` used to signify an intentional lack of value..

  • What do you mean by REST API?

    • A REST API (Representational State Transfer Application Programming Interface) is a standardized way for software applications to communicate over the internet. It uses standard HTTP methods and URLs to perform actions like retrieving, creating, updating, or deleting data from a server. REST APIs are widely used for building web and mobile applications due to their simplicity and scalability.
